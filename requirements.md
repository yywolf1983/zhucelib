# Android 注册系统 - 需求文档

## 一、概述

本系统包含两个核心组件:

| 组件 | 类型 | 用途 |
|---|---|---|
| `registration-lib` | Android Library(AAR) | 注册库,引入到任意 Android 项目,置于主界面之前提供注册控制 |
| `keygen-app` | Android Application(APK) | 注册机,用于生成激活码 |

## 二、核心需求

### 2.1 加密方案

- **算法**:RSA-2048 + SHA256withRSA 签名
- **安全模型**:非对称加密,私钥与公钥分离
  - **公钥**:编译时写死在注册库源码中,源码可公开
  - **私钥**:注册机从本地文件选择(.pem/.der),不存储在任何 App 内
- **协议**:挑战-响应模式
  - 客户机生成随机安装码(含设备指纹 + 随机数)
  - 注册机用私钥签名生成激活码(含购买时长设置信息)
  - 客户机用公钥验签,绑定设备+本次挑战

### 2.2 时间控制

| 配置项 | 说明 | 范围 |
|---|---|---|
| `trialDays` | 试用天数 | 0~65535,0=不试用 |
| `validDays`(激活码) | 购买时长(天) | 0~65535,0=永久 |

### 2.3 注册框弹出时机

| 模式 | 说明 |
|---|---|
| `FIRST_LAUNCH` | 试用期第一次启动就弹出注册/试用框 |
| `ON_EXPIRY` | 试用期不弹,到期后才弹出注册框 |

### 2.4 到期后行为

| 模式 | 说明 |
|---|---|
| `NAG_ONLY` | 只弹提示框,用户可关闭继续使用(可配合业务级断言自行限制功能) |
| `BLOCK` | 限制功能,必须重新激活才能进入主界面 |

### 2.5 到期后每次启动行为

- 试用结束后或购买时长结束后,**每次打开 App 都弹出提示框**
- 提示框内容区分「试用到期」与「授权到期」

### 2.6 绝对控制权

库必须对所控制的软件源码有绝对控制权,不能轻易跳过:

1. **Application 级拦截**:提供 `RegGateApplication` 基类,在每次 Activity 启动时强制校验
2. **生命周期守卫**:注册 `ActivityLifecycleCallbacks`,未注册则终止进程
3. **业务级断言**:提供 `ensureRegistered()` 方法,宿主可在关键业务方法前调用
4. **入口 GateActivity**:置于主界面之前
5. **运行时重新验签**:每次启动重新验证激活码,不从缓存信任

### 2.7 隐写配置

设置可以隐写到注册码中，被 RSA 签名保护：

| 配置项 | 隐写位置 | 取值 |
|---|---|---|
| `trialDays` | flags 字节 bit 0-1 | 0=3天, 1=7天, 2=14天, 3=无试用 |
| `promptTiming` | flags 字节 bit 2 | 0=FIRST_LAUNCH, 1=ON_EXPIRY |
| `expireBehavior` | flags 字节 bit 3 | 0=BLOCK, 1=NAG_ONLY |

**安全特性：**
- 配置写入激活码，被 RSA 签名保护
- 即使看到源码，也无法篡改或伪造配置
- 未激活时使用默认配置，激活后使用激活码中的配置
- 激活码不能太复杂：Crockford Base32 编码，自动分组便于输入

### 2.8 注册码易于输入

- **编码**:Crockford Base32(排除易混淆字符 I/L/O/U)
- **分组**:自动用 `-` 分隔,复制时自动去除
- **容错**:O→0、I/L→1 自动转换

### 2.9 简单加入与剥离

**加入**(3 处改动):
1. 引入 `registration-lib` 依赖
2. Application 继承 `RegGateApplication` 或调用 `installLifecycleGuard()`
3. `RegGateConfig.init()` 初始化配置(只需设置公钥和主界面类)
4. Manifest 把启动入口改为 `RegistrationGateActivity`

**剥离**(反向操作):
1. 移除依赖
2. Application 改回继承 `Application`
3. 移除 `RegGateConfig.init()` 调用
4. Manifest 改回原启动 Activity

## 三、数据格式

### 3.1 安装码(客户机 → 注册机)

```
格式: Base32(deviceId[12] || nonce[8])
长度: 32 字符,8 组×4 字符
示例: ABCD-EFGH-IJKL-MNOP-QRST-UVWX-YZ01-2345
```

### 3.2 激活码(注册机 → 客户机)

```
格式: Base32(validDays[2 BE] || issuedDay[4 BE] || sig[256])
长度: 448 字符,89 组×5 字符(含一个 3 字符尾组)
签名消息: deviceId[12] || nonce[8] || validDays[2 BE] || issuedDay[4 BE]
```

### 3.3 时间字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `issuedDay` | uint32 | 自 1970-01-01 起的天数 |
| `validDays` | uint16 | 有效天数,0=永久 |
| `expiryMs` | int64 | 到期时间戳,0=永久 |

## 四、状态机

```
┌─────────────────────────────────────────────────────────────┐
│                        启动                                 │
└──────────────────────────────┬──────────────────────────────┘
                               ▼
                    ┌───────────────────┐
                    │   检查激活状态     │
                    └─────────┬─────────┘
                              │
         ┌────────────────────┼────────────────────┐
         ▼                    ▼                    ▼
   ┌─────────────┐    ┌─────────────┐    ┌───────────────┐
   │   LICENSED  │    │  TRIALING   │    │ EXPIRED/      │
   │  已激活     │    │  试用期内   │    │ NEED_REGISTER │
   └──────┬──────┘    └──────┬──────┘    └───────┬───────┘
          │                  │                   │
          ▼                  │                   ▼
    直接进入主界面            │           进入注册界面 / 弹提示框
                             │
              ┌───────────────┼───────────────┐
              ▼                               ▼
      promptTiming =                     promptTiming =
      FIRST_LAUNCH                        ON_EXPIRY
              │                               │
              ▼                               ▼
   首次启动弹试用框                    到期后弹注册框
   然后进入主界面                    然后进入主界面
```

## 五、安全特性

| 特性 | 说明 |
|---|---|
| 设备绑定 | 激活码与设备指纹绑定,换机失效 |
| 随机挑战 | 每次安装码随机,无法预先批量造码 |
| 加密存储 | 激活码用 EncryptedSharedPreferences(AES-256)存储 |
| 密钥分离 | 私钥永不进入 App 内部,源码可公开 |
| 运行时校验 | 每次启动重新验签,防篡改 |
| 进程终止 | 未注册时终止进程,无法绕过 |

## 六、非功能性需求

- **最小 SDK**:Android 5.0(API 21)
- **无第三方加密依赖**:纯 Java 原生 RSA API
- **易于集成**:3~4 处改动即可
- **易于剥离**:反向操作即可完全移除
- **源码可公开**:安全核心在于私钥,不在源码中
