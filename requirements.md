# Android 注册系统 - 需求文档

## 一、概述

本系统包含三个核心组件：

| 组件 | 类型 | 用途 |
|---|---|---|
| `registration-lib` | Android Library(AAR) | 注册库，提供注册验证和试用控制 |
| `keygen-app` | Android Application(APK) | 注册机，使用私钥签名生成激活码 |
| `demo-app` | Android Application(APK) | 演示应用，展示集成方式 |

## 二、核心需求

### 2.1 加密方案

- **算法**: RSA-2048 + SHA256withRSA 签名
- **安全模型**: 非对称加密，私钥与公钥分离
  - **公钥**: 编译时内置到注册库资源文件中，源码可公开
  - **私钥**: 注册机从本地文件选择(.pem)，不存储在任何 App 内
- **协议**: 挑战-响应模式
  - 客户机生成安装码(设备指纹 + 随机数)
  - 注册机用私钥签名生成激活码
  - 客户机用公钥验签，绑定设备+本次挑战

### 2.2 时间控制

| 配置项 | 说明 | 范围 |
|---|---|---|
| `trialDays`(库配置) | 试用天数 | 0~65535, 0=不试用 |
| `validDays`(激活码) | 购买时长(天) | 0~65535, 0=永久 |

### 2.3 注册框弹出时机

| 模式 | 说明 |
|---|---|
| `FIRST_LAUNCH` | 首次启动弹出注册/试用框 |
| `ON_EXPIRY` | 到期后弹出注册框 |
| `EVERY_LAUNCH` | 每次启动都弹出注册/试用框 |

### 2.4 到期后行为

| 模式 | 说明 |
|---|---|
| `NAG_ONLY` | 只弹提示框，用户可关闭继续使用 |
| `BLOCK` | 限制功能，必须激活才能进入主界面 |

### 2.5 绝对控制权

库必须对所控制的软件有绝对控制权：

1. **Application 级拦截**: 提供 `RegGateApplication` 基类
2. **生命周期守卫**: 注册 `ActivityLifecycleCallbacks`，未注册则终止进程
3. **业务级断言**: 提供 `ensureRegistered()` 方法
4. **入口 GateActivity**: 置于主界面之前
5. **运行时重新验签**: 每次启动重新验证激活码

### 2.6 试用配置

试用配置（试用天数、弹框时机、到期行为）写死在注册库中：

- 默认试用 7 天
- 默认每次启动弹框
- 默认到期限制功能

宿主可通过 `RegGateConfig` 覆盖默认配置。

### 2.7 注册码易于输入

- **编码**: Crockford Base32(排除易混淆字符 I/L/O/U)
- **分组**: 自动用 `-` 分隔，复制时自动去除
- **容错**: O→0、I/L→1 自动转换

### 2.8 简单加入与剥离

**加入**(3 处改动):
1. 引入 `registration-lib` 依赖
2. Application 中调用 `RegGateConfig.init().mainActivity().build()`
3. 调用 `new RegistrationManager(this).installLifecycleGuard(this)`
4. Manifest 把启动入口改为 `RegistrationGateActivity`

**剥离**(反向操作):
1. 移除依赖
2. 移除 `RegGateConfig.init()` 和 `installLifecycleGuard()` 调用
3. Manifest 改回原启动 Activity

### 2.9 注册机特性

- 私钥文件选择后自动记住位置（SharedPreferences + 持久化 URI 权限）
- 无需每次启动都选择私钥

## 三、数据格式

### 3.1 安装码(客户机 → 注册机)

```
格式: Base32(deviceId[12] || nonce[8])
长度: 32 字符, 8 组×4 字符
示例: ABCD-EFGH-IJKL-MNOP-QRST-UVWX-YZ01-2345
```

### 3.2 激活码(注册机 → 客户机)

```
格式: Base32(validDays[2 BE] || issuedDay[4 BE] || sig[256])
长度: 448 字符, 89 组×5 字符
签名消息: deviceId[12] || nonce[8] || validDays[2 BE] || issuedDay[4 BE]
```

### 3.3 时间字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `issuedDay` | uint32 | 自 1970-01-01 起的天数 |
| `validDays` | uint16 | 有效天数, 0=永久 |
| `expiryMs` | int64 | 到期时间戳, 0=永久 |

## 五、安全特性

| 特性 | 说明 |
|---|---|
| 设备绑定 | 激活码与设备指纹绑定，换机失效 |
| 随机挑战 | 每次安装码随机，无法预先批量造码 |
| 加密存储 | 激活码用 EncryptedSharedPreferences(AES-256)存储 |
| 密钥分离 | 私钥永不进入 App 内部，源码可公开 |
| 运行时校验 | 每次启动重新验签，防篡改 |
| 进程终止 | 未注册时终止进程，无法绕过 |

## 六、非功能性需求

- **最小 SDK**: Android 5.0(API 21)
- **无第三方加密依赖**: 纯 Java 原生 RSA API
- **易于集成**: 3~4 处改动即可
- **易于剥离**: 反向操作即可完全移除
- **源码可公开**: 安全核心在于私钥
